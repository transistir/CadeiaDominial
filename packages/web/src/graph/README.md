# `packages/web/src/graph/`

Graph rendering pipeline for the web app. The pipeline is intentionally linear
and every step has a single, narrow responsibility:

```
graph JSON  →  validateGraph  →  layoutGraph  →  toReactFlow  →  <ReactFlow>
   (unknown)   (GraphJson)      (LayoutedGraph)  (React Flow   (rendered to
                                                        nodes/edges)     DOM)
```

## Files

| File | Role | Input → Output |
|------|------|----------------|
| `types.ts` | Shared `GraphJson` / `GraphNode` / `GraphEdge` types. | — |
| `validateGraph.ts` | **Trust boundary.** Accepts `unknown`, validates shape + integrity, returns a strongly-typed `GraphJson`. | `unknown` → `GraphJson` |
| `layoutGraph.ts` | Deterministic dagre LR layout. Trust contract: caller has already validated. | `GraphJson` → `LayoutedGraph` |
| `toReactFlow.ts` | Adapts the layouted graph into React Flow's `Node`/`Edge` shapes. | `LayoutedGraph` → `{ nodes, edges }` |
| `GraphPreview.tsx` | The React Flow renderer. Memoizes the pipeline on the input ref. | `unknown` → rendered React Flow |
| `fixtures/basic-graph.json` | Canonical 3-node / 2-edge fixture (source → process → output). | — |
| `index.ts` | Public barrel. | — |

## Design rules

1. **Validation happens once, at the boundary.** `validateGraph` accepts
   `unknown` so it can defend against any input (JSON file, API response, user
   upload). Downstream functions trust the typed `GraphJson` it returns.
2. **Layout is deterministic.** Same input → same positions. This makes the
   e2e screenshot test meaningful (it can be regenerated byte-stable).
3. **No animation / no randomness** in the rendering path. React Flow is
   configured `nodesDraggable={false}`, `animated: false`, etc. so screenshots
   are reproducible.
4. **Memoize the whole pipeline.** `GraphPreview` runs the pipeline inside a
   `useMemo` keyed on the input ref, so a re-render with the same graph is
   free.
5. **No `any`.** The pipeline is strict-TS throughout. If you find yourself
   reaching for `as SomeType` to silence a complaint, prefer a real
   validation in `validateGraph`.

## Adding a new fixture

Drop a `.json` file in `fixtures/`, then import it from a route or test. The
file shape is exactly `GraphJson` (see `types.ts`). Anything that fails the
shape will throw at the first render — that's intentional.

## Adding a new layout algorithm

`layoutGraph` is a pure function over `GraphJson`. A new layout (e.g. force
graph, radial) is a new function with the same signature, then a small
switch in `GraphPreview` if both are exposed. Don't try to make `layoutGraph`
itself pluggable — the contract is "dagre LR for now".
