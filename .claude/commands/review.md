# ADD Review Phase (Codex)

Use Codex to review the implementation against the spec and plan.

## Your Task

### 1. Prerequisites Check

Find the most recent issue directory and verify:
- `04-plan.md` is committed to git
- Implementation commits exist (check git log for recent commits)
- If no implementation: Display error "No implementation found. Run /dev first."

### 2. Gather Review Context

Collect:
- Plan: `.project/issues/ISSUE-[id]/04-plan.md`
- Spec: `.project/issues/ISSUE-[id]/02-spec.md`
- Changed files: Use `git diff` to get all changes since plan was committed

### 3. Consult Codex for Review

Use the `mcp__codex-bridge__consult_codex_with_stdin` tool with:
- **stdin_content**: The git diff output
- **prompt**: A comprehensive review request including:
  - The spec acceptance criteria
  - The implementation plan steps
  - Request for thorough code review
- **directory**: Current working directory
- **format**: "json"
- **timeout**: 600 (10 minutes)

Example prompt structure:
```
Review this implementation against the spec and plan:

SPECIFICATION:
[paste spec acceptance criteria]

PLAN:
[paste plan steps]

Please review the code changes for:
1. Alignment with specification - Does it implement all acceptance criteria?
2. Adherence to plan - Were all planned steps completed correctly?
3. Code quality - Is the code clean, maintainable, and well-structured?
4. Security - Any security concerns or vulnerabilities?
5. Performance - Any obvious performance issues?
6. Testing - Are tests adequate and do they cover edge cases?

Provide:
- Summary of changes reviewed
- Issues found (critical, high, medium, low priority)
- Recommendations for improvement
- Approval status (approve/revise/reject)
```

### 4. Save Review

Write Codex's review to a new file: `.project/issues/ISSUE-[id]/05-review.md`

### 5. Display Completion

```
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
Review Phase Complete (Codex)
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

Artifact: .project/issues/ISSUE-[id]/05-review.md
Generated by: Codex

Review Status: [Approve/Revise/Reject]
Issues Found: [count by priority]

Next Steps:
1. Review Codex's feedback
2. If revisions needed: Make changes and run /review again
3. If approved: Commit 05-review.md and run /qa
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
```

## Notes

- Codex has 10-minute timeout for review
- Review can be run multiple times if revisions are needed
- Only proceed to /qa after Codex review is satisfactory
- Commit the review artifact when satisfied
