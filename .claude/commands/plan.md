# ADD Plan Phase (Codex)

Use Codex to generate an implementation plan for the approved spec and tests.

## Your Task

### 1. Prerequisites Check

Find the most recent issue directory and verify:

- `03-tests.md` is committed to git
- If not committed: Display error "Tests phase not approved. Commit 03-tests.md first."

### 2. Load Artifacts

Read these files from `.project/issues/ISSUE-[id]/`:

- `00-context.md` - Scope and boundaries
- `02-spec.md` - Specification with acceptance criteria
- `03-tests.md` - Test specification

### 3. Consult Codex for Plan

Use the `mcp__codex-bridge__consult_codex` tool with:

- **query**: A prompt that includes:
  - The context, spec, and tests content
  - Request for a step-by-step implementation plan
  - Emphasis on: alignment with spec, atomic steps, explicit file references
- **directory**: Current working directory
- **format**: "json"
- **timeout**: 600 (10 minutes)

Example query structure:

```
Create an implementation plan for this issue:

CONTEXT:
[paste 00-context.md]

SPECIFICATION:
[paste 02-spec.md]

TESTS:
[paste 03-tests.md]

Generate a detailed step-by-step implementation plan that:
1. Implements all acceptance criteria from the spec
2. Ensures all tests will pass
3. Specifies exact files to change
4. Provides atomic, verifiable steps

Format as markdown with sections:
- Alignment Check
- Steps (with file paths, changes, verification)
- Files Changed Summary
- Verification Checklist
```

### 4. Save Plan

Write Codex's response to `.project/issues/ISSUE-[id]/04-plan.md`

### 5. Display Completion

```
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
Plan Phase Complete (Codex)
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

Artifact: .project/issues/ISSUE-[id]/04-plan.md
Generated by: Codex

Next Steps:
1. Review the implementation plan
2. Commit 04-plan.md when satisfied (git commit = approval)
3. Run /dev to implement the plan
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
```

## Notes

- Codex has 10-minute timeout for complex planning
- If Codex times out, consider breaking the issue into smaller sub-issues
- Review Codex's plan carefully before approving (committing)
